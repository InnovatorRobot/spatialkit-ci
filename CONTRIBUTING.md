# Contributing to SpatialRender CI Lab

Thank you for your interest in contributing! This is an educational project, but we welcome improvements.

## Development Setup

1. Follow the [Quick Start guide](README.md#quick-start) in the README
2. Make your changes
3. Run the full test suite before submitting

## Code Style

- Follow the existing code style (2-space indentation)
- Use `clang-format` for C++ files
- Keep functions focused and well-documented

## Testing

All contributions should:
- Pass unit tests: `cd build && ctest`
- Pass visual regression tests (if rendering-related)
- Maintain or improve benchmark performance

## Pull Request Process

1. Create a feature branch
2. Make your changes
3. Ensure CI passes
4. Submit a PR with a clear description

## Areas for Contribution

- Additional test scenes
- Performance optimizations
- Documentation improvements
- CI/CD enhancements
- Cross-platform compatibility fixes
